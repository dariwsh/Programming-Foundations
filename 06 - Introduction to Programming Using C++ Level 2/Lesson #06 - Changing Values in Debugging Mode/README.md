# 📘 C++ Debugging Practice - تغيير قيمة المتغير أثناء التنفيذ

## 📌 نظرة عامة
المشروع ده بيشرح واحدة من أهم تقنيات الـ Debugging في C++:

> تغيير قيمة المتغير أثناء تنفيذ البرنامج (Runtime Debugging)

التقنية دي بتسمح لك تختبر سيناريوهات مختلفة بسرعة من غير ما تعدل الكود أو تعيد تشغيل البرنامج كل مرة.

---

## 🎯 أهداف التعلم

- فهم كيفية تتبع القيم داخل الذاكرة
- استخدام Breakpoints بذكاء
- تعديل قيم المتغيرات أثناء التنفيذ
- تحليل اللوب خطوة بخطوة
- تتبع استدعاء الفنكشن

---

## 🧠 الكود

```cpp
#include<iostream>
using namespace std;

int MySum(int a, int b)
{
	int s = 0;
	s = a + b;
	return s;
}

int main()
{
	int arr1[5] = { 200,100,50,25,30 };
	int a, b, c;

	a = 10;
	b = 20;

	a++;
	++b;

	c = a + b;

	cout << a << endl;
	cout << b << endl;
	cout << c << endl;

	for (int i = 1; i <= 5; i++)
	{
		cout << i << endl;
		a = a + a * i;
	}

	c = MySum(a, b);

	cout << c;

	return 0;
}
```

---

## 🧪 الفكرة الأساسية

أثناء تشغيل البرنامج في وضع **Debug Mode**:

يمكنك إيقاف التنفيذ عند نقطة معينة (Breakpoint)،  
ثم تعديل قيمة أي متغير يدويًا،  
وملاحظة تأثير ذلك على باقي البرنامج.

---

## 🔍 نقطة التجربة (Breakpoint)

ضع Breakpoint عند هذا السطر:

```cpp
a = a + a * i;
```

---

## ▶️ خطوات التنفيذ

1. تشغيل البرنامج باستخدام Debug (F5)
2. البرنامج يتوقف عند الـ Breakpoint
3. مراقبة القيم في:
   - Autos Window
4. تعديل قيمة المتغير `a`
5. استكمال التنفيذ

---

## 🧠 تتبع التنفيذ (Tracing)

### قبل الدخول في اللوب:

```
a = 11
b = 21
c = 32
```

---

### داخل اللوب:

#### أول دورة:
```
i = 1
a = 22
```

#### ثاني دورة:
```
i = 2
a = 66
```

---

## 🔄 تجربة التغيير اليدوي

### الحالة الطبيعية:

```
a = 22
i = 2
→ a = 66
```

---

### بعد التعديل أثناء Debug:

تقوم بتغيير:

```
a = 100
```

---

### النتيجة:

```
a = 100 + (100 × 2)
a = 300
```

---

## ⚠️ ملاحظات مهمة

- التعديل لا يغيّر الكود الأصلي
- التعديل مؤقت (Runtime فقط)
- يتم إلغاء التغيير عند إعادة تشغيل البرنامج
- مفيد جدًا لاختبار الحالات المختلفة بسرعة

---

## 🚀 لماذا هذا مهم؟

- يساعدك على اكتشاف الأخطاء المنطقية (Logic Errors)
- يوفر وقت إعادة تشغيل البرنامج
- يعطيك فهم أعمق لسلوك البرنامج
- يستخدمه المحترفون يوميًا في Debugging

---

## 🧩 الأدوات المستخدمة

- Breakpoints
- Autos Window
- Step Into (F11)
- Continue (F5)
- تتبع القيم داخل اللوب
- تحليل استدعاء الفنكشن

- Debugging ليس فقط لإصلاح الأخطاء
