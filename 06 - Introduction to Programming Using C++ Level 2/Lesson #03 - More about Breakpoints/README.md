# 🚀 C++ Debugging - Breakpoints Mastery

> Learn how to debug like a pro using Breakpoints in C++

---

## 🎯 Why this matters?

Debugging is one of the **most important skills** for any programmer.

If you don't know how your code runs → you can't fix it.

👉 Breakpoints help you:

* See what really happens inside your program
* Track variables step by step
* Find bugs faster

---

## 🧠 What is a Breakpoint?

A **Breakpoint** is a point where the program stops during execution.

💡 Think of it like:

> "Pause the program and check what's going on"

---

## ⚡ Quick Example

```cpp id="fl96qb"
#include <iostream>
using namespace std;

int main()
{
    int x = 5;
    int y = 10;

    int sum = x + y; // 🔴 Breakpoint here

    cout << sum;

    return 0;
}
```

---

## 🔍 What happens when it stops?

When execution stops:

* You can see values of variables
* You can check logic step by step
* You can understand errors

---

## 🛠 Debugging Tools (Super Important)

| Tool        | What it does    |
| ----------- | --------------- |
| ▶ Continue  | Run program     |
| ⏭ Step Over | Go to next line |
| ⏬ Step Into | Enter function  |
| 🔼 Step Out | Exit function   |

---

## 🔥 Pro Tips

* Don't spam breakpoints everywhere ❌
* Put them in **important logic parts** ✅
* Always use Debug mode
* Watch variables while stepping

---

## 🧩 Practice Like a Pro

Try this:

* Add breakpoint inside a loop
* Watch how values change
* Debug a function step by step
